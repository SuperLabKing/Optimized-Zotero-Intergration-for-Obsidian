# Zotero Integration v2.1.0 设计文档

## 概述

v2.1.0 版本聚焦于插件配置界面的美化和数据模型规范化，包括：
1. 数据基建重构：废弃冗余字段、统一四字 label、实现 YAML 强类型规范
2. 视图层 UI 重构：修复交互 bug、优化布局比例、引入现代化卡片样式

## 架构概览

### 分层设计

**数据层** (`src/bbt/` 和 `src/locale/`)
- 字段定义和提取逻辑
- YAML 强类型输出规范
- 中英文 UI 文案

**视图层** (`src/settings/settings.tsx` 和 `styles.css`)
- 属性映射设置的交互修复
- 布局比例优化
- 现代化卡片样式

两层通过 `PropertyMapping` 类型和 `SMART_FIELD_OPTIONS` 数组连接，修改数据层不会破坏视图层渲染逻辑。

## 数据层重构

### 字段废弃

从 `src/bbt/smartExtractors.ts` 删除以下字段：
- `journal` - 出版物
- `journal_platform_smart` - 期刊/平台
- `publisher_smart` - 出版社

删除对应的提取函数：
- `extractJournal`
- `extractJournalPlatformSmart`
- `extractPublisherSmart`

### 字段重命名（四字原则）

更新 `SMART_FIELD_OPTIONS` 数组的 label：

| 字段名 | 旧 label | 新 label |
|--------|---------|---------|
| `title_smart` | 标题 (包含超链接) | 文献标题 |
| `authors_smart` | 作者 (智能处理 et al. 与通讯) | 文献作者 |
| `impact_factor_smart` | 文库编目 (提取 IF 值) | 影响因子 |
| `year` | 日期 (提取年份) | 发表年份 |
| `collections_path` | 文库和分类 (多级路径) | 文库分类 |
| `type_tags_smart` | 标签：文献类型 (#) | 文献类型 |
| `status_tags_smart` | 标签：阅读状态 (/) | 阅读状态 |
| `priority_tags_smart` | 标签：星标优先级 (⭐) | 星标评级 |
| `extra_translation` | 其他 (提取附加翻译) | 附加信息 |

### YAML 强类型规范

修改 `src/bbt/templateEngine.ts` 的 `recordToYaml` 函数：

**单值字段**（输出为 `key: value`）：
- `title_smart`
- `impact_factor_smart`
- `year`
- `extra_translation`

**列表字段**（强制输出为 `key: [item1, item2]`）：
- `authors_smart`
- `collections_path`
- `type_tags_smart`
- `status_tags_smart`
- `priority_tags_smart`

实现逻辑：
```typescript
const SINGLE_VALUE_FIELDS = ['title_smart', 'impact_factor_smart', 'year', 'extra_translation'];

// 在 recordToYaml 中
if (Array.isArray(value)) {
  // 列表字段
  lines.push(`${key}: [${items.join(', ')}]`);
} else if (typeof value === 'number') {
  // 数字字段（IF）
  const formatted = Number.isInteger(value) ? value.toFixed(1) : value.toString();
  lines.push(`${key}: ${formatted}`);
} else {
  // 字符串字段
  lines.push(`${key}: ${yamlEscape(String(value))}`);
}
```

## 视图层重构

### 交互 Bug 修复

#### 1. 删除按钮失效

**问题**：删除映射时页面跳动，scrollTop 丢失

**修复** (`src/settings/settings.tsx:_renderPropertyMappings`):
```typescript
.addExtraButton((btn) =>
  btn
    .setIcon('trash')
    .setTooltip(t('settings.template.deleteMapping'))
    .onClick((event) => {
      event.stopPropagation();
      event.preventDefault();
      
      const container = wrapper.parentElement;
      const scrollTop = container?.scrollTop || 0;
      
      const updated = [...(this.plugin.settings.propertyMappings || [])];
      updated.splice(i, 1);
      this.plugin.settings.propertyMappings = updated;
      this.debouncedSave();
      this._renderPropertyMappings(container);
      
      if (container) {
        container.scrollTop = scrollTop;
      }
    })
);
```

#### 2. 拖拽重影（如果存在）

如果代码中有拖拽功能，添加：
```typescript
element.addEventListener('dragstart', (e) => {
  element.classList.add('is-dragging');
  e.dataTransfer.setDragImage(element, 0, 0);
});

element.addEventListener('dragend', () => {
  element.classList.remove('is-dragging');
});
```

## 错误处理

### 数据层边界情况

1. **旧配置兼容**：
   - 用户配置中的废弃字段不会导致错误
   - 废弃字段不再出现在下拉菜单中
   - 已保存的废弃字段映射会被忽略

2. **类型转换**：
   - 单值字段返回数组时，取第一个元素
   - 列表字段返回单值时，包装成单元素数组
   - 空值/null/undefined 统一跳过

3. **YAML 输出**：
   - 影响因子始终输出为 `.0` 格式（如 `5.0`）
   - 字符串自动转义特殊字符
   - 数组为空时不输出该字段

### 视图层边界情况

1. **空状态**：
   - 映射列表为空时显示"添加映射"按钮
   - 删除最后一个映射后保留容器和按钮

2. **拖拽功能**：
   - 如果代码中不存在拖拽，跳过拖拽相关修复
   - 拖拽样式通过 CSS 类名控制，不影响其他功能

3. **滚动恢复**：
   - 删除操作后恢复 scrollTop
   - 容器不存在时跳过滚动恢复

## 风险与缓解

### 风险 1：废弃字段导致用户配置丢失

**缓解**：
- 废弃字段不会导致错误，只是不再显示
- 用户可以手动删除旧映射，添加新映射
- 在 Release Notes 中说明变更

### 风险 2：强类型转换破坏现有数据

**缓解**：
- 充分测试各种数据类型
- 保持向后兼容，空值跳过
- 在测试环境验证后再部署

### 风险 3：UI 改动影响其他功能

**缓解**：
- 只修改属性映射相关代码
- 保持其他 Tab 代码不变
- 完整回归测试
